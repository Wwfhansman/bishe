# 语音实验执行与论文表格模板

## 1. 文档目的

本文档用于统一说明以下内容：

1. 如何执行当前项目中的语音实验脚本
2. 如何得到降噪、STT、打断三类实验结果
3. 如何把结果整理为毕业论文中的表格、图表和结论

本文档与以下文档配套：

- `docs/adaptive-barge-in-design.md`
- `docs/barge-in-experiment-guide.md`
- `docs/噪声抑制与语音增强设计文档.md`

当前仓库已具备以下实验脚本：

- `scripts/eval_noise_suppression.py`
- `scripts/compare_stt_noise_suppression.py`
- `scripts/eval_barge_in.py`
- `scripts/run_voice_experiments.py`

其中，`scripts/run_voice_experiments.py` 是推荐主入口。

---

## 2. 推荐实验输入

### 2.1 单段语音样本

用于降噪与 STT 对比。

要求：

- `wav`
- `16kHz`
- 单声道
- `16bit PCM`

建议内容：

- 厨房环境下的一段人声提问
- 同时存在背景噪声

### 2.2 打断清单

用于打断实验。

格式：

- JSONL

样例：

```json
{"path":"samples/interrupt_01.wav","label":"interrupt","expected_interrupt_ms":820,"tts_offset_ms":0}
{"path":"samples/echo_only_01.wav","label":"no_interrupt","tts_offset_ms":0}
```

## 3. 推荐主脚本

### 3.1 统一执行脚本

推荐使用：

```bash
python scripts/run_voice_experiments.py
```

这个脚本可以统一完成：

1. 降噪前后能量对比
2. 降噪前后 STT 对比
3. 原始音频与增强音频的打断检测对比
4. 导出 JSON 和 Markdown 摘要

### 3.2 同时执行全部实验

示例：

```bash
python scripts/run_voice_experiments.py ^
  --wav samples\\question.wav ^
  --manifest samples\\barge_manifest.jsonl ^
  --enhanced-wav outputs\\question_enhanced.wav ^
  --output-json outputs\\voice_experiment.json ^
  --output-markdown outputs\\voice_experiment.md ^
  --noise-floor 900 ^
  --noise-alpha 0.12 ^
  --spectral-alpha 0.18 ^
  --min-gain 0.4 ^
  --over-subtraction 1.2 ^
  --speech-margin 2.0
```

执行后将得到：

1. 增强后的音频文件
2. 完整 JSON 结果
3. 可直接放进论文草稿的 Markdown 表格摘要

## 4. 分项脚本说明

### 4.1 仅看降噪效果

```bash
python scripts/eval_noise_suppression.py sample.wav --output-wav enhanced.wav
```

用途：

1. 查看能量前后变化
2. 输出增强音频试听

### 4.2 仅看 STT 前后对比

```bash
python scripts/compare_stt_noise_suppression.py sample.wav --output-wav enhanced.wav
```

用途：

1. 对比 `raw_stt.final_text`
2. 对比 `enhanced_stt.final_text`
3. 对比 `first_partial_latency_ms`

### 4.3 仅看打断对比

```bash
python scripts/eval_barge_in.py manifest.jsonl --compare-noise-suppression
```

用途：

1. 对比原始音频与降噪音频上的打断效果
2. 输出 `raw.summary`
3. 输出 `enhanced.summary`

## 5. 推荐实验顺序

为了让论文结构清晰，建议按照下面顺序做实验。

### 第一步：降噪效果确认

目的：

- 先确认增强模块没有明显破坏人声

方法：

1. 输出增强音频
2. 主观试听
3. 记录输入能量、输出能量和能量比

### 第二步：STT 对比

目的：

- 验证降噪是否有助于识别

方法：

1. 使用同一段 `wav`
2. 比较 `raw_stt.final_text` 与 `enhanced_stt.final_text`
3. 统计若干样本的正确率或人工评分

### 第三步：打断对比

目的：

- 验证降噪是否有助于打断检测

方法：

1. 使用 JSONL 清单
2. 同时输出原始和增强两组结果
3. 比较 `precision`、`recall`、`f1`、`false_interrupt_rate`

- `raw.summary`
- `enhanced.summary`

重点指标：

- `precision`
- `recall`
- `f1`
- `false_interrupt_rate`
- `miss_rate`
- `avg_detect_latency_ms`

## 9. 结果分析模板

### 9.1 降噪模块分析模板

实验结果表明，轻量级噪声抑制模块能够在一定程度上压制厨房环境中的背景噪声，同时较好保留目标人声信号。与原始音频相比，增强后音频的背景能量有所下降，说明该模块对稳态噪声具有一定抑制效果。

### 9.2 STT 改善分析模板

在语音识别实验中，增强音频在若干噪声样本上取得了更稳定的识别结果，说明语音前端增强能够减少背景噪声对识别模块的干扰。但在部分样本中，增强处理也可能带来轻微语音失真，因此需要在降噪强度与语音保真度之间进行权衡。

### 9.3 打断检测改善分析模板

在打断检测实验中，增强音频版本在误打断率和漏检率之间取得了更优平衡，表明语音前端增强有助于降低背景噪声对打断判定的影响。特别是在风扇声和水流声场景下，增强后音频的检测稳定性优于原始音频。

## 10. 写论文时的建议

建议将整套实验写成以下逻辑：

1. 先分析厨房噪声问题
2. 再介绍语音前端增强策略
3. 然后展示 STT 前后对比
4. 最后展示打断检测前后对比

这样结构会比较自然，因为它体现的是：

`前端增强 -> 识别改善 -> 打断改善`

这比把几组实验零散摆出来更容易说服老师。

---

## 11. 当前推荐做法

在当前阶段，最推荐的执行方式是：

1. 先准备一段单句 `wav`
2. 再准备一份打断清单 `manifest.jsonl`
3. 用 `scripts/run_voice_experiments.py` 一次性输出 JSON 和 Markdown
4. 把 Markdown 结果拷到论文草稿中，再人工润色

这样你后续实验会非常省时间。
