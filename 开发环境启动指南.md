# 厨房助手开发环境启动指南

当前版本以前后端分离方式开发：

- 后端：`FastAPI + WebSocket`
- 客户端：`Flutter`
- Web 页面：仅用于本地联调

## 1. 首次准备

### 安装 Python 依赖

```powershell
cd C:\Users\34222\Desktop\bishe
.\venv\Scripts\Activate.ps1
pip install -r backend\requirements.txt
```

### 安装 Flutter 依赖

```powershell
cd C:\Users\34222\Desktop\bishe\app-flutter
flutter pub get
```

### 配置后端环境变量

在项目根目录准备 `.env`，至少填好：

```bash
ARK_API_KEY=你的ARK_API_KEY
ARK_MODEL_ID=doubao-1-5-pro-32k-250115
ARK_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
```

## 2. 日常启动

### 终端 1：启动后端

```powershell
cd C:\Users\34222\Desktop\bishe
.\venv\Scripts\Activate.ps1
uvicorn backend.api.server:app --host 127.0.0.1 --port 8000 --reload
```

### 终端 2：按调试场景启动 Flutter

Windows 桌面：

```powershell
cd C:\Users\34222\Desktop\bishe\app-flutter
flutter run --dart-define=API_BASE_URL=http://127.0.0.1:8000 --dart-define=WS_BASE_URL=ws://127.0.0.1:8000/ws/voice
```

Android 模拟器：

```powershell
cd C:\Users\34222\Desktop\bishe\app-flutter
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:8000 --dart-define=WS_BASE_URL=ws://10.0.2.2:8000/ws/voice
```

Android 真机：

先启动 ngrok：

```powershell
cd C:\Users\34222\Desktop\bishe
.\ngrok.exe http 8000
```

再把 `Forwarding` 地址带入 Flutter：

```powershell
cd C:\Users\34222\Desktop\bishe\app-flutter
flutter run --dart-define=API_BASE_URL=https://你的地址.ngrok-free.dev --dart-define=WS_BASE_URL=wss://你的地址.ngrok-free.dev/ws/voice
```

## 4. 当前版本约定

- 不再通过手改 `constants.dart` 切换地址，统一使用 `--dart-define`。
- `frontend/index.html` 是调试页面，不是主客户端。
- 本地数据库文件是 `voice_assistant.db`，旧的 `chat.db` 已不再使用。
- `backend/rag/chroma_db/` 是本地生成物，可以重新构建。
