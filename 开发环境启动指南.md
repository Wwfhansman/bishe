# 🍳 厨房助手 AI — 开发环境启动指南

## 一、首次配置（只需做一次）

### 1. 安装 Python 依赖
```powershell
cd C:\Users\34222\Desktop\bishe
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 2. 配置 ngrok（内网穿透，用于真机调试）
1. 注册免费账号：https://dashboard.ngrok.com/signup
2. 在 Dashboard 获取 Authtoken
3. 执行：
```powershell
.\ngrok.exe config add-authtoken 你的authtoken
```

### 3. 微信开发者工具设置
- 打开微信开发者工具 → **设置** → **代理** → 选择 **直连网络**
- **详情** → **本地设置** → 勾选 **不校验合法域名、web-view(业务域名)、TLS版本以及HTTPS证书**

## 二、每日开发启动流程

### 方式一：一键启动（推荐）
双击项目根目录下的 `start_dev.bat`，它会自动：
- 启动后端服务器（端口 8001）
- 启动 ngrok 内网穿透

### 方式二：手动启动
打开两个终端窗口：

**终端 1 — 后端服务器：**
```powershell
cd C:\Users\34222\Desktop\bishe
.\venv\Scripts\Activate.ps1
uvicorn backend.api.server:app --host 127.0.0.1 --port 8001
```

**终端 2 — ngrok 隧道（真机调试时才需要）：**
```powershell
cd C:\Users\34222\Desktop\bishe
.\ngrok.exe http 8001
```

**终端 3 — 前端（App-mobile）开发者工具：**
```powershell
cd C:\Users\34222\Desktop\bishe\app-mobile
npx expo start
```
第二步：在终端里按下 a 键，Expo 会自动在您的 Android 模拟器上打开这个 App。
第三步：您只要在编辑器（如 VS Code）里保存（Ctrl+S）代码，模拟器上的画面就会瞬间自动刷新出最新效果！您刚才看到的那个红色报错界面，只要改对了代码一保存，它也会自动变成正常的界面。
2. 特殊情况（安装了新的、带底层硬件功能的原生包）
只有当您使用 npm install 安装了包含底层原生 C/Java 代码的库（比如我们刚才安装的音频流插件，或者涉及到蓝牙、相机权限的新底层插件），您才需要重新对 Android 工程进行编译。 此时您需要执行（因为您的环境配置，需要带上临时 Java 变量）：

powershell
$env:JAVA_HOME="C:\Program Files\Android\Android Studio\jbr"; npx expo run:android

## 三、更新前端连接地址

所有网络地址统一在 `wxapp/utils/config.js` 中管理：

```javascript
const ENV = 'local';   // ← 开发者工具模拟器用 'local'
const ENV = 'ngrok';   // ← 真机调试用 'ngrok'
```

### ngrok 地址更新
每次重启 ngrok 后，它会分配一个新的公网地址（免费版每次不同），格式如：
```
https://xxxxx-xxxxx.ngrok-free.dev
```

把这个地址更新到 `config.js` 的 ngrok 配置中：
```javascript
ngrok: {
    API_BASE: 'https://新地址.ngrok-free.dev/api',
    WS_BASE: 'wss://新地址.ngrok-free.dev/ws/voice'
}
```

## 四、测试方式

| 测试方式 | config.js 中 ENV 值 | 需要 ngrok？ | 说明 |
|---------|---------------------|-------------|------|
| 开发者工具模拟器 | `'local'` | ❌ 不需要 | 直连 127.0.0.1:8001 |
| 真机调试 / 真机预览 | `'ngrok'` | ✅ 需要 | 手机通过公网访问 |

### 开发者工具测试
1. 确保 `config.js` 中 `ENV = 'local'`
2. 编译运行即可（Ctrl+B）

### 真机调试测试
1. 确保 ngrok 正在运行
2. 复制 ngrok 给的公网地址，粘贴到 `config.js`
3. 确保 `config.js` 中 `ENV = 'ngrok'`
4. 在微信开发者工具中点击 **真机调试**，手机扫码

## 五、常见问题

### Q: 请求报 `ERR_CONNECTION_REFUSED`
- 检查后端是否已启动（终端有没有显示 `Uvicorn running on ...`）
- 检查 `config.js` 中的 ENV 和地址是否正确
- 检查微信开发者工具的代理设置是否为"直连网络"

### Q: ngrok 报 `authentication failed`
- 执行 `.\ngrok.exe config add-authtoken 你的authtoken`

### Q: 开发者工具里语音识别没反应
- 这是开发者工具的**已知限制**，模拟器的录音格式与真机不同
- 语音识别功能请在**真机调试**模式下测试

### Q: 真机无法连接
- 确认 ngrok 终端窗口没有关闭
- 确认 `config.js` 中 ENV 为 `'ngrok'` 且地址已更新
